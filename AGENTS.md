## What Is This?

ZeroClaw Mission Control is a **zero-trust agent orchestration platform** that
runs specialized AI agents (researcher, coder, orchestrator) behind a secure
gateway with token-based authentication, network isolation, and Redis-backed
session management.

## Security Model

This repo follows **zero-trust principles**:

| Layer | Protection |
|-------|-----------|
| Secrets | `.env` gitignored; tokens generated by `openssl rand` |
| Network | Bridge network; Redis internal-only (no host port) |
| Containers | Non-root, read-only FS, no-new-privileges |
| Resources | CPU/memory limits on all containers |
| Supply chain | All GHA actions SHA-pinned; Dependabot enabled |
| Detection | `detect-secrets` pre-commit hook; weekly security audit |
| Rotation | `rotate-secrets.sh` for credential cycling |

## When Working on This Repo

### DO ‚úÖ
- Run `pre-commit run --all-files` before committing
- Add SPDX headers to all new files
- SHA-pin any new GitHub Action references
- Include health checks on new Docker services
- Set resource limits on new containers
- Use `${ENV_VAR}` for any credential reference
- Sign commits (`git commit -s`)

### DON'T ‚ùå
- Hardcode any secret, token, password, or API key
- Expose Redis port to the host
- Run containers as root
- Use `--no-verify` to bypass pre-commit
- Use floating tags (`@v4`) for GitHub Actions
- Add `chmod 777` or `sudo` to scripts
- Commit `.env` files

## Agent Modification Guide

When editing agent personas (`agents/*/agent.md` or `agents/*/tools.md`):

1. **Maintain separation of concerns** ‚Äî Researcher researches, coder codes,
   orchestrator delegates
2. **Least privilege** ‚Äî Only grant tools the agent strictly needs
3. **Document changes** ‚Äî Update the agent's constraints section
4. **Test delegation** ‚Äî Verify handoff paths still work
